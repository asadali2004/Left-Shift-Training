ItTechGenie — C# Scenario Based Questions

Topics: Generics, Action, Func, Predicate | Difficulty: Medium (5) + Hard (5)

Instructions

Each question is scenario-based and expects you to implement using Generics + Action + Func + Predicate (not necessarily all four in every question, but the required topics are clearly mentioned per question).

Generic <T> Action<...> Func<...> Predicate<T>

Medium Level — 5 Questions

Q1. Generic Filter for HR Portal Medium

Scenario: In an HR portal, you maintain lists of employees, contractors, and interns. You need a single reusable method to filter any list based on a condition (e.g., salary > 50K, experience > 2 years, department = "IT").
Requirements:
  • Create a generic method Filter<T>(IEnumerable<T> items, Predicate<T> rule)
  • Demonstrate with an Employee list: filter employees with Salary ≥ 50000
  • Print the filtered employees
Topics: Generics, Predicate, Action (printing/logging)

Q2. Audit Logger Wrapper for Any Operation Medium

Scenario: You want a common audit wrapper that logs "START" and "END" for different operations like bill calculation, report generation, or file upload.
Requirements:
  • Create RunWithAudit<TResult>(string title, Func<TResult> work, Action<string> logger)
  • Use it to calculate an electricity bill with tax and return the final amount
  • Logger prints time-stamped messages
Topics: Generics, Func, Action

Q3. Generic Mapper for API DTO Conversion Medium

Scenario: Your backend returns domain objects (e.g., Product) but the UI expects DTOs (e.g., ProductDto). You need a generic mapper to convert lists efficiently.
Requirements:
  • Create Map<TSource, TResult>(IEnumerable<TSource> items, Func<TSource, TResult> selector)
  • Convert Product to ProductDto and compute a field like DisplayPrice
  • Print the mapped DTO list
Topics: Generics, Func, Action

Q4. Customer Eligibility Rules (Multiple Predicates) Medium

Scenario: A finance app approves a customer for a loan if multiple conditions are met (e.g., age range, salary threshold, credit score, no blacklist).
Requirements:
  • Create ValidateAll<T>(T obj, params Predicate<T>[] rules)
  • Define at least 3 rules for a Customer object and validate it
  • Print "Approved" or "Rejected"
Topics: Generics, Predicate, Action

Q5. Batch Processor for Notifications Medium

Scenario: A notification service sends messages to different recipients. You want a reusable batch processor that can perform an action on each item and also optionally filter items.
Requirements:
  • Create ProcessBatch<T>(IEnumerable<T> items, Predicate<T> filter, Action<T> action)
  • Use it to send a message only to users with IsActive = true
  • Action prints: "Sent to: {user}"
Topics: Generics, Predicate, Action

Hard Level — 5 Questions

Q6. Predicate Composition Engine (AND / OR Rules) Hard

Scenario: In an e-commerce platform, you apply multiple filters dynamically: in-stock AND category match AND price range. Sometimes filters are OR-based as well. You need a rule composition engine.
Requirements:
  • Create generic methods: And<T>(Predicate<T> a, Predicate<T> b) and Or<T>(...)
  • Combine 3+ rules for Product and filter a list
  • Print matched products
Topics: Generics, Predicate, Action

Q7. Generic Pipeline: Validate → Transform → Execute Hard

Scenario: In a billing system, you receive an Invoice. You must validate it, compute final amount, then save/print the result. This must be reusable for other types too.
Requirements:
  • Create Process<T, TResult>(T input, Predicate<T> validator, Func<T, TResult> transformer, Action<TResult> executor)
  • Use it for Invoice: validate (amount > 0), transform (add tax), execute (print/store)
  • Handle validation failure with exception or error message
Topics: Generics, Predicate, Func, Action

Q8. Generic Cache Wrapper (GetOrAdd) for Expensive Computations Hard

Scenario: A risk engine calculates a customer risk score (expensive). You must cache results by key to avoid recalculations.
Requirements:
  • Create Cache<TKey, TValue> with method GetOrAdd(TKey key, Func<TKey, TValue> factory)
  • Demonstrate by computing risk score for same customer id twice
  • Print "Computing..." only once and show cached value next time
Topics: Generics, Func, Action

Q9. Generic Retry Engine for Unstable Operations Hard

Scenario: Payment gateway calls may fail due to network issues. Implement a reusable retry engine that retries an operation and logs each attempt.
Requirements:
  • Create Retry(Action work, int maxAttempts, Action<string> logger)
  • Simulate failure for first 2 attempts and success on 3rd attempt
  • Log each attempt and final status
Topics: Action, Generics (optional extension), Func (optional extension)

Q10. Generic Report Builder (Filter + Sort + Project) Hard

Scenario: You are building a reusable report generator. For any type T, you should be able to:
  • Filter items using Predicate
  • Sort items using Func (key selector)
  • Project items to a printable string using Func
  • Print using Action
Requirements:
  • Create BuildReport<T, TKey>(IEnumerable<T> items, Predicate<T> filter, Func<T, TKey> orderBy, Func<T, string> projector, Action<string> printer)
  • Use it for Employee list: filter salary > 50K, order by name, project to "Name - Salary"
  • Print the report lines
Topics: Generics, Predicate, Func, Action
