XMLHttpRequest â€” The Engine Behind AJAX

The browser's built-in tool to talk to servers

ğŸ”§ What is XMLHttpRequest (XHR)?

XMLHttpRequest is a built-in browser object that lets JavaScript send and receive data from a server without reloading the page. jQuery's $.ajax() and the modern fetch() are both built on top of this same idea.

XHR is like a phone call your code makes. It dials the server, waits for an answer, and when the server picks up and responds â€” your JavaScript code handles whatever comes back.

The raw XHR code (so you understand what's happening under the hood):

javascript â€” XMLHttpRequest the raw way
// Step 1: Create the XHR object
var xhr = new XMLHttpRequest();

// Step 2: Configure it â€” method (GET/POST), URL, async(true=don't freeze page)
xhr.open("GET", "https://some-api.com/data.xml", true);

// Step 3: Tell it what format to expect back
xhr.responseType = "document";  // "document" = parse XML automatically

// Step 4: What to do when the server responds
xhr.onload = function() {
    if (xhr.status === 200) {
        // SUCCESS â€” xhr.responseXML is the parsed XML
        var xml = xhr.responseXML;
        console.log("Got XML!", xml);
    } else {
        console.log("Error: " + xhr.status);
    }
};

// Step 5: Handle network failures
xhr.onerror = function() {
    console.log("Network error!");
};

// Step 6: Actually SEND the request
xhr.send();  // nothing happens until you call this!
Notice: nothing happens until xhr.send(). Steps 1â€“5 are just setup. Step 6 fires the actual request.

jQuery $.ajax() â€” The Easy Wrapper

Same thing as XHR but much cleaner to write

ğŸ’š Why use $.ajax()?

jQuery's $.ajax() wraps the messy XHR code above into a clean, readable format. Under the hood it still creates an XMLHttpRequest â€” it just handles all the boring parts for you.

javascript â€” Same request with jQuery $.ajax()
$.ajax({
    type:        'GET',                     // HTTP method
    url:         'https://some-api.com/data.xml', // where to send
    dataType:    'xml',                    // tell jQuery: expect XML back

    // runs when server replies successfully
    success: function(xmlResponse) {
        console.log('Got XML!', xmlResponse);
        // xmlResponse is already a parsed XML Document
    },

    // runs if something goes wrong
    error: function(xhr, status, err) {
        console.log('Failed: ' + err);
    }
});

Same result, far less code. This is why everyone uses jQuery for AJAX.

JSON.stringify() & JSON.parse()

Converting between JavaScript objects and text

â“ Why do we need to "stringify"?

When you send data over the internet, you can only send text (strings). But in JavaScript you work with objects. JSON.stringify() converts your object into a text string so it can travel over HTTP. On the other side, JSON.parse() converts it back to an object.

It's like packing a 3D model into a flat box (stringify) to ship it, then assembling it on the other side (parse). You can't ship assembled furniture easily â€” you flatten it first.

Reading structured data from tags

XML looks similar to HTML but it's data-focused, not display-focused. Every piece of data is wrapped in custom tags you define. APIs use XML to send structured information.

javascript â€” Parsing XML with getElementsByTagName
// xmlDoc = the parsed XML document (comes from AJAX response)

// Get ALL <entry> tags â€” returns a list (like an array)

// Loop through each entry
for (var i = 0; i < entries.length; i++) {

    var entry = entries[i];

    // Get text inside a specific tag
    var trackName = entry
        .getElementsByTagName("trackName")[0]  // [0] = first match
        .textContent;                          // .textContent = the text inside

    var artist = entry
        .getElementsByTagName("artistName")[0]
        .textContent;

    console.log(trackName + " by " + artist);
}
